# AI 应用平台配色方案

## 1. 设计目标

本方案面向企业级 AI 应用聚合平台，覆盖平台主页、应用中心、登录页与管理后台。整体视觉应体现稳定、可信、专业和高效，避免过度装饰、过强渐变和高饱和色堆叠。

设计原则：

- 以深蓝色作为品牌主色，建立企业级、科技感和可信赖的第一印象。
- 搭配中性色承载大面积背景、文字、边框和分割关系，保证长时间使用的舒适度。
- 使用少量辅助冷色丰富层级，避免整个系统只重复一种蓝色。
- 红色、黄色仅用于异常、预警、风险、删除、不可逆操作和冲突提示。
- 门户主页可以更有品牌感，管理后台必须更克制，强调信息识别、操作效率和问题定位。

## 2. 核心色板

### 2.1 品牌主色

| 用途 | 色值 | 使用说明 |
| --- | --- | --- |
| 主品牌深蓝 | `#0B2A4A` | 平台 Logo 字色、后台侧边栏深色版本、首页核心标题、关键品牌识别 |
| 主操作蓝 | `#145DA0` | 主按钮、当前导航、关键链接、选中态边线 |
| 交互亮蓝 | `#2F80C8` | hover、focus、可点击图标、轻量强调 |
| 信息浅蓝 | `#EAF4FC` | 选中背景、信息提示底色、浅色标签背景 |

### 2.2 辅助冷色

| 用途 | 色值 | 使用说明 |
| --- | --- | --- |
| 科技青蓝 | `#27A7C9` | 首页局部强调、数据卡图标、非主要 CTA 辅助强调 |
| 钢蓝灰 | `#4F6F8F` | 次级标题、后台图标、表格辅助信息 |
| 靛蓝 | `#263B73` | 首页深色区块、登录页品牌侧背景、重要但非操作类强调 |

### 2.3 中性色

| 用途 | 色值 | 使用说明 |
| --- | --- | --- |
| 页面背景 | `#F5F7FA` | 后台主工作区、应用中心页面背景 |
| 门户浅背景 | `#F7FAFD` | 首页白色内容区背景 |
| 卡片背景 | `#FFFFFF` | 卡片、表格容器、弹窗、下拉面板 |
| 次级背景 | `#F0F4F8` | 表头、筛选栏、浅色分区 |
| 边框 | `#DCE5EE` | 卡片、输入框、表格、弹窗边界 |
| 强边框 | `#B9C8D6` | 输入框 focus 前的可见边界、禁用按钮边框 |
| 主文字 | `#172635` | 标题、表格主内容、正文高优先级文本 |
| 次级文字 | `#5E7184` | 说明文字、描述、表格辅助字段 |
| 弱提示文字 | `#8A9AAA` | placeholder、禁用文字、空状态说明 |

### 2.4 状态色

| 状态 | 色值 | 使用说明 |
| --- | --- | --- |
| 成功绿 | `#2E7D59` | 启用、成功保存、正常状态 |
| 成功浅底 | `#EAF7F0` | 成功标签背景 |
| 风险红 | `#C2413B` | 删除、失败、禁用、异常 |
| 风险浅底 | `#FFF0EF` | 删除确认、错误提示背景 |
| 预警黄 | `#D68A00` | 权限冲突、风险提示、清空操作 |
| 预警浅底 | `#FFF7E8` | Warning Alert、风险说明背景 |

红色和黄色不得用于普通强调、品牌装饰、常规按钮或普通统计图标。

## 3. 平台主页配色

### 3.1 首页背景

主页第一屏建议使用深蓝品牌背景，强化平台定位：

- Hero 背景：`#0B2A4A`
- Hero 深色辅助层：`#102F52`
- 局部冷色光感：`#145DA0` / `#27A7C9`
- Hero 文本：`#FFFFFF`
- Hero 副标题：`rgba(255,255,255,0.78)`

建议使用低对比的径向光感或大面积深蓝渐变，但不要使用多色彩虹渐变，也不要加入过多漂浮装饰元素。

### 3.2 首页内容区

- 内容区背景：`#F7FAFD`
- 分区背景交替：`#FFFFFF` / `#F5F7FA`
- 首页标题：`#0B2A4A`
- 首页说明文字：`#5E7184`
- 分区标题下划线或小面积强调：`#145DA0`

### 3.3 首页应用卡片

| 元素 | 色值 |
| --- | --- |
| 卡片背景 | `#FFFFFF` |
| 卡片边框 | `#DCE5EE` |
| 卡片 hover 边框 | `#2F80C8` |
| 卡片 hover 阴影 | `rgba(11,42,74,0.12)` |
| 图标容器背景 | `#EAF4FC` |
| 图标颜色 | `#145DA0` |
| 标题 | `#172635` |
| 描述 | `#5E7184` |

卡片顶部可使用 2px 到 3px 的品牌蓝强调线，但不建议每张卡片使用不同高饱和颜色。

### 3.4 首页按钮

| 类型 | 背景 | 文字 | 边框 | 使用场景 |
| --- | --- | --- | --- | --- |
| 主按钮 | `#145DA0` | `#FFFFFF` | `#145DA0` | 立即体验、进入应用中心 |
| 主按钮 hover | `#0B2A4A` | `#FFFFFF` | `#0B2A4A` | 鼠标悬停 |
| 次按钮 | `transparent` | `#145DA0` | `#B9C8D6` | 查看详情、了解能力 |
| 深色区次按钮 | `rgba(255,255,255,0.08)` | `#FFFFFF` | `rgba(255,255,255,0.42)` | Hero 区辅助操作 |

## 4. 应用中心配色

应用中心偏向工作台和检索场景，应比首页更克制。

- 页面背景：`#F5F7FA`
- 顶部筛选容器：`#FFFFFF`
- 搜索框背景：`#FFFFFF`
- 搜索框边框：`#DCE5EE`
- 搜索框 focus：`#2F80C8`
- 分类选中背景：`#EAF4FC`
- 分类选中文字：`#145DA0`
- 排序下拉菜单背景：`#FFFFFF`
- 下拉菜单 hover：`#F0F4F8`

应用卡片继续沿用首页卡片体系，但卡片间距应更规整，减少强装饰，提高扫描效率。

## 5. 登录页配色

登录页建议采用左右分区：

- 品牌侧背景：`#0B2A4A`
- 品牌侧辅助块：`#102F52`
- 登录表单背景：`#FFFFFF`
- 页面背景：`#F5F7FA`
- 输入框边框：`#DCE5EE`
- 输入框 focus：`#2F80C8`
- 登录主按钮：`#145DA0`
- 忘记密码 / 找回账号链接：`#145DA0`
- 错误提示：`#C2413B`
- 错误提示背景：`#FFF0EF`

验证码、账号锁定、密码错误等风险信息只使用红色体系，不使用黄色或蓝色弱化风险。

## 6. 管理后台配色

后台管理包含应用管理、用户管理、角色管理、组织管理和监控总览，应以稳定、克制、可读为核心。

### 6.1 后台整体背景

| 区域 | 色值 | 说明 |
| --- | --- | --- |
| 主工作区背景 | `#F5F7FA` | 大面积背景，降低视觉疲劳 |
| 内容卡片背景 | `#FFFFFF` | 表格、筛选栏、表单容器 |
| 顶部栏背景 | `#FFFFFF` | 面包屑、当前用户、退出入口 |
| 分割线 | `#DCE5EE` | 顶栏、侧栏和内容分隔 |

### 6.2 后台侧边栏

推荐深色侧边栏版本，增强企业系统识别：

| 状态 | 背景 | 文字 / 图标 |
| --- | --- | --- |
| 侧边栏背景 | `#0B2A4A` | - |
| 一级菜单文字 | - | `rgba(255,255,255,0.82)` |
| 一级菜单 hover | `#102F52` | `#FFFFFF` |
| 当前菜单 | `#145DA0` | `#FFFFFF` |
| 当前菜单左边线 | `#27A7C9` | - |
| 分组说明文字 | - | `rgba(255,255,255,0.56)` |

如果需要更轻量的后台，可使用白色侧边栏：

- 侧边栏背景：`#FFFFFF`
- 菜单文字：`#40566A`
- hover 背景：`#F0F4F8`
- 选中背景：`#EAF4FC`
- 选中文字：`#0B2A4A`
- 选中左边线：`#145DA0`

两种方案只能选一种，避免深浅侧栏混用。

### 6.3 后台页面标题区

- 页面标题：`#172635`
- 页面说明：`#5E7184`
- 标题区底部分割线：`#DCE5EE`
- 主操作按钮：`#145DA0`
- 次操作按钮：白底 + `#B9C8D6` 边框

标题区应保持 24px 到 32px 下边距，不建议把标题、筛选栏和表格挤在同一视觉层级。

## 8. 字体与层级建议

字体建议：

```text
-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", "Segoe UI", "Source Han Sans CN", sans-serif
```

页面层级：

| 场景 | 字号 | 字重 | 色值 |
| --- | --- | --- | --- |
| 首页 Hero 标题 | 48px - 64px | 700 | `#FFFFFF` |
| 首页分区标题 | 28px - 32px | 700 | `#0B2A4A` |
| 后台页面标题 | 22px - 24px | 700 | `#172635` |
| 卡片标题 | 16px - 18px | 600 | `#172635` |
| 表格正文 | 13px - 14px | 400 | `#172635` |
| 辅助说明 | 12px - 13px | 400 | `#5E7184` |

后台不要使用过大的标题字号，重点保持信息密度适中、行高舒适、字段易扫描。

## 9. 页面应用建议

### 9.1 平台主页

- 第一屏使用深蓝背景承载品牌标题和主按钮。
- 应用矩阵使用白色卡片，图标以浅蓝底承载，避免卡片过彩。
- 平台价值区使用浅灰白背景，突出文字层级，不使用红黄装饰色。

### 9.2 应用管理

- 表格为主，筛选栏白底，表头浅灰。
- 新增应用使用主按钮蓝。
- 删除应用使用风险红。
- 授权人数、启用状态可使用信息蓝和成功绿标签。

### 9.3 用户管理

- 用户头像可使用主操作蓝或钢蓝灰，不同用户不需要随机彩色头像。
- 账号禁用使用风险红标签。
- 重置密码属于风险操作，可使用预警黄或二次确认弹窗。

### 9.4 角色管理

- 权限树使用浅灰边界和浅蓝选中态。
- 权限冲突、缺失依赖、清空权限必须使用预警黄。
- 移除权限、删除角色使用风险红。

### 9.5 组织管理

- 企业、部门列表保持中性色表格。
- 权限下放矩阵中，选中态用浅蓝底和主蓝勾选。
- 删除企业/部门必须使用风险红确认。

### 9.6 监控总览

- 摘要卡图标使用主蓝、青蓝、钢蓝灰组合，不使用红黄表示普通指标。
- 只有异常、告警、失败率、风险应用才使用红色或黄色。
- 空状态使用弱提示文字和浅灰图标。

## 11. 禁用规则

- 禁止将红色、黄色用于普通按钮、普通图标、品牌装饰和普通统计卡。
- 禁止所有页面重复使用同一个蓝色值，应通过深蓝、主蓝、亮蓝、浅蓝建立层级。
- 禁止后台表格使用高饱和大面积背景。
- 禁止在后台同时使用深色侧栏和大量彩色卡片，避免工作台显得杂乱。
- 禁止用低对比文字承载关键字段，例如账号、角色、状态、权限名称。
