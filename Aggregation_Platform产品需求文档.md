## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | AI 应用聚合平台 |
| 项目目录 | Aggregation_Platform |
| 技术现状 | React 18 + Vite 5 + TypeScript 5 + Ant Design 5 |

## 2. 产品背景

AI 应用聚合平台面向企业内部 AI 能力统一入口建设，当前项目已实现门户首页雏形，包括顶部导航、品牌 Hero 区、AI 应用矩阵、分类筛选、应用搜索、应用卡片跳转和平台价值展示。平台后续需从“AI 应用展示入口”升级为“企业级 AI 应用运营与治理平台”，补齐账号登录、角色权限、Token 配额、系统监控与运维能力。

平台核心目标是将企业内外部 AI 应用统一纳管，围绕用户身份、应用权限、资源用量、运行状态和审计追踪建立完整闭环，满足企业级安全、可控、可运维要求。

## 3. 当前项目已实现功能

### 3.1 门户首页

当前首页由 `TopNav`、`HeroBanner`、`AppMatrix`、`ValueSection` 组成，具备以下能力：

- 顶部导航：支持首页、解决方案、帮助中心导航入口。
- 应用搜索：支持根据应用名称或描述搜索，命中后滚动定位到对应应用卡片。
- 品牌展示：Hero 区展示平台名称、平台定位和主操作按钮。
- 应用矩阵：以卡片形式展示 AI 应用，点击后跳转到外部或内部应用地址。
- 应用分类：支持按全部应用、研发提效、硬件设计、办公协同分类筛选。
- 平台价值：展示平台覆盖、安全合规、稳定保障、持续迭代等价值点。

### 3.2 当前应用范围

当前数据配置中已规划的 AI 应用包括：

| 应用 | 类型 | 说明 |
| --- | --- | --- |
| 知识助手 | 办公协同 | 企业知识库检索与问答 |
| 会议纪要 | 办公协同 | 音频转写、会议摘要、待办提取 |
| 代码生成 | 研发提效 | 基于自然语言生成代码 |
| 代码审查 | 研发提效 | 代码风险识别与优化建议 |
| 电路审查 | 硬件设计 | 电路设计合规与风险检查 |
| 元器件库 | 硬件设计 | 元器件查询、选型与参数对比 |
| 文档生成 | 办公协同 | 生成技术文档、报告、方案等 |
| 文档审查 | 办公协同 | 文档校对、格式、逻辑和合规审查 |

### 3.3 当前能力边界

当前项目尚未实现登录鉴权、后台管理、角色权限、Token 配额管理、系统监控、日志查询、审计追踪等企业级管理能力。后续建设应在现有门户首页基础上扩展为完整前后台一体化平台。

## 4. 产品定位与目标

### 4.1 产品定位

AI 应用聚合平台是企业内部统一 AI 能力门户和运营治理平台，为员工提供一站式 AI 应用入口，为管理员提供权限、配额、监控、日志、审计等管理能力。

### 4.2 建设目标

- 统一入口：聚合企业内多个 AI 应用，提供搜索、分类、跳转和推荐能力。
- 统一身份：通过登录页和认证体系识别用户身份、企业、部门和角色。
- 统一权限：基于角色和权限树控制不同用户可访问的模块、功能和操作。
- 统一配额：按企业、部门、用户组、个人管理 Token 配额和使用情况。
- 统一监控：集中展示系统资源、服务状态、任务队列、API 调用和日志。
- 统一审计：记录权限变更、配额调整、登录行为、关键操作和异常事件。

## 5. 用户角色

| 角色 | 说明 | 关键权限 |
| --- | --- | --- |
| 普通用户 | 企业内部 AI 应用使用者 | 登录平台、访问授权应用、查看个人用量 |
| 部门管理员 | 部门级管理人员 | 查看本部门用户和用量，管理部门内部分配项 |
| 企业管理员 | 企业级管理员 | 管理本企业用户、角色、权限、配额和日志 |
| 超级管理员 | 平台级管理员 | 跨企业管理、全局配置、系统监控、全局审计 |
| 运维人员 | 系统运行维护人员 | 查看监控、日志、告警、服务状态和任务队列 |

## 6. 信息架构

## 7. 功能需求

### 7.1 登录页

#### 7.1.1 页面目标

登录页用于完成企业用户身份认证，并作为用户进入平台的第一道安全入口。页面风格应保持简洁、稳重、企业级，支持品牌自定义。

#### 7.1.2 视觉与布局

- 页面采用企业级简洁视觉风格，避免复杂装饰。
- 支持自定义背景图，背景图可由企业管理员在后台配置。
- 支持企业 Logo、平台名称、品牌色配置。
- 登录表单居中布局，在桌面端展示为居中卡片或居中面板。
- 移动端登录表单需自适应屏幕宽度，保证输入和按钮可正常操作。
- 品牌色需作用于主按钮、焦点态、标签选中态和关键链接。

#### 7.1.3 认证方式

登录表单需通过标签页区分不同认证方式：

| 认证方式 | 字段 | 说明 |
| --- | --- | --- |
| 账号密码登录 | 账号、密码、验证码可选 | 默认认证方式 |
| 邮箱登录 | 邮箱、验证码 | 用于邮箱验证码登录 |
| 企业单点登录 | 企业标识或自动跳转 | 对接企业 SSO 时启用 |

#### 7.1.4 忘记密码

- 登录页提供“忘记密码”入口。
- 点击后打开独立弹窗，不跳离当前登录页。
- 流程简化为：输入邮箱 -> 获取验证码 -> 设置新密码 -> 完成。
- 支持邮箱找回，需校验邮箱格式、验证码有效期和密码强度。
- 修改成功后提示用户重新登录。

#### 7.1.5 找回账号

- 登录页提供“找回账号”入口。
- 点击后打开独立弹窗。
- 用户输入企业邮箱后，系统向邮箱发送账号信息或找回链接。
- 若邮箱对应多个账号，需提示联系企业管理员或展示脱敏账号列表。

#### 7.1.6 异常提示

- 账号不存在、密码错误、验证码错误、账号禁用、企业禁用等状态需明确提示。
- 连续失败达到阈值后需触发安全策略，例如验证码、短时锁定或管理员通知。
- 所有错误提示应避免泄露敏感信息。

### 7.2 门户首页与应用矩阵

#### 7.2.1 首页展示

首页延续当前已实现结构，并补充登录后的个性化能力：

- 顶部导航保留当前项目中可用的首页、应用矩阵搜索、分类筛选、应用卡片跳转和平台价值展示能力。
- 顶部导航展示企业名称、通知入口和右上角用户入口。
- 配额中心、监控运维、权限管理等管理类功能统一放入右上角用户功能下拉菜单，不作为顶部导航一级菜单直接展示。
- Hero 区展示平台定位、企业品牌和关键入口。
- 应用矩阵展示用户有权限访问的 AI 应用。
- 未授权应用默认不展示；如业务需要，也可展示但置灰并提示申请权限。

#### 7.2.2 应用搜索与筛选

- 支持按应用名称、描述、分类搜索。
- 支持按分类筛选，包括研发提效、硬件设计、办公协同等。
- 支持按最近使用、推荐应用、热门应用排序。

#### 7.2.3 应用访问控制

- 用户点击应用时，系统需校验登录态、角色权限、企业范围和 Token 配额状态。
- 若无权限，提示“暂无访问权限，请联系管理员”。
- 若配额用尽，提示“当前 Token 配额不足，请联系管理员调整配额”。
- 所有应用访问行为需写入操作日志。

#### 7.2.4 右上角用户功能下拉

右上角用户功能下拉用于承载个人能力和管理能力，减少顶部导航复杂度，保持首页大气简洁。

下拉菜单建议包含：

| 菜单项 | 可见角色 | 说明 |
| --- | --- | --- |
| 个人中心 | 全部登录用户 | 查看个人资料、所属企业、部门、角色 |
| 我的 Token 用量 | 全部登录用户 | 查看个人 Token 配额、使用趋势和剩余额度 |
| 权限管理 | 企业管理员、超级管理员 | 进入用户管理、角色管理、权限配置和权限审计 |
| 配额中心 | 部门管理员、企业管理员、超级管理员 | 进入配额配置、配额监控和配额日志 |
| 监控运维 | 运维人员、超级管理员 | 进入监控总览、资源看板和系统日志 |
| 操作日志 | 企业管理员、超级管理员、运维人员 | 查看授权范围内的操作日志 |
| 退出登录 | 全部登录用户 | 退出当前账号 |

交互要求：

- 下拉菜单从用户头像或用户名点击触发。
- 菜单项需根据当前用户权限动态展示，无权限项不展示。
- 管理类菜单项可使用分组分隔，例如“个人功能”和“管理功能”。
- 若用户仅为普通用户，下拉菜单应保持轻量，仅展示个人中心、我的 Token 用量和退出登录。
- 点击菜单项后进入对应后台页面，后台页面可使用统一管理布局。

### 7.3 角色管理权限

#### 7.3.1 页面目标

角色管理页用于集中管理企业内角色、权限分配和权限变更审计，确保不同用户只能访问被授权的模块、功能和操作。

#### 7.3.2 页面布局

角色管理页采用“角色列表 + 权限配置侧边栏”的布局：

- 左侧或主区域为角色列表。
- 右侧为权限配置侧边栏。
- 选择某一角色后，侧边栏展示该角色的权限树、已选权限、冲突提示和保存按钮。

#### 7.3.3 角色列表

角色列表字段包括：

| 字段 | 说明 |
| --- | --- |
| 角色名称 | 角色显示名称 |
| 角色编码 | 系统内唯一编码 |
| 所属企业 | 企业管理员仅可见本企业 |
| 用户数量 | 当前绑定该角色的用户数 |
| 状态 | 启用、停用 |
| 创建人 | 创建角色的操作人 |
| 更新时间 | 最近更新时间 |
| 操作 | 查看、编辑、复制、停用、删除 |

角色管理需支持新增、编辑、复制、停用、删除、搜索、筛选和分页。

#### 7.3.4 权限树

权限按“模块 - 功能 - 操作”三级树形展示：

```text
权限
├─ 应用中心
│  ├─ 应用列表
│  │  ├─ 查看
│  │  ├─ 访问
│  │  └─ 申请权限
│  └─ 应用配置
│     ├─ 查看
│     ├─ 新增
│     ├─ 编辑
│     └─ 删除
├─ 权限管理
│  ├─ 用户管理
│  │  ├─ 查看
│  │  ├─ 新增
│  │  ├─ 编辑
│  │  ├─ 删除
│  │  └─ 重置密码
│  └─ 角色管理
│     ├─ 查看
│     ├─ 新增
│     ├─ 编辑
│     ├─ 删除
│     └─ 分配权限
├─ Token 配额管理
│  ├─ 配额配置
│  │  ├─ 查看
│  │  ├─ 调整
│  │  └─ 审批
│  └─ 配额日志
│     ├─ 查看
│     └─ 导出
└─ 系统监控与运维
   ├─ 监控总览
   │  └─ 查看
   └─ 系统日志
      ├─ 查看
      └─ 导出
```

#### 7.3.5 权限分配

- 权限项采用可视化树形勾选。
- 支持父子级联选择。
- 支持“全选”和“反选”。
- 已选权限需高亮显示。
- 支持搜索权限节点，搜索命中时自动展开相关路径。
- 保存前展示权限变更摘要，包括新增权限、移除权限和影响范围。

#### 7.3.6 权限审计

权限审计采用时间轴展示权限变更记录，记录内容包括：

| 字段 | 说明 |
| --- | --- |
| 操作时间 | 权限变更发生时间 |
| 操作人 | 执行变更的用户 |
| 操作对象 | 被修改的角色或用户 |
| 变更内容 | 新增、删除或修改的权限 |
| 变更原因 | 操作人填写或系统自动生成 |
| 操作结果 | 成功、失败、部分成功 |

时间轴支持按角色、操作人、时间范围和变更类型筛选。

#### 7.3.7 权限边界

- 企业管理员仅能操作本企业的角色和用户。
- 企业管理员不可查看、编辑、删除其他企业数据。
- 超级管理员可跨企业操作角色、用户和权限。
- 超级管理员操作跨企业数据时需记录更高等级审计日志。
- 系统内置超级管理员角色不可被企业管理员修改。

#### 7.3.8 权限冲突检测

系统在分配权限时需自动检测权限冲突，并阻止保存。

示例冲突规则：

| 冲突权限 | 冲突原因 | 处理方式 |
| --- | --- | --- |
| 删除用户 + 仅查看用户 | 操作权限与只读权限互斥 | 阻止保存并提示冲突项 |
| 跨企业管理 + 企业内管理 | 管理范围冲突 | 提示选择一种管理范围 |
| 停用角色 + 分配角色 | 角色状态操作与分配操作冲突 | 阻止保存 |
| 导出日志 + 无日志查看 | 缺少前置权限 | 提示先勾选查看权限 |

冲突提示需展示冲突节点、冲突原因和建议处理方式。

#### 7.3.9 权限实时生效

- 权限变更保存成功后需实时生效。
- 用户无需重新登录即可应用新的权限结果。
- 前端应在路由切换、菜单渲染、接口请求前刷新或校验权限。
- 后端接口必须以服务端权限结果为准，不能仅依赖前端控制。

### 7.4 Token 配额管理

#### 7.4.1 页面目标

Token 配额管理用于控制企业、部门、用户组和个人维度的 AI 使用量，避免资源超用，并提供使用趋势、预警和审计能力。

#### 7.4.2 配额层级

配额按以下层级展示和配置：

```text
企业
└─ 部门
   └─ 用户组
      └─ 个人
```

配额支持从上级向下级分配，下级总额不得超过上级可分配额度。若启用共享池策略，则需明确共享池额度、优先级和用尽后的处理策略。

#### 7.4.3 配额配置页

配额配置页采用“树形导航 + 配置面板”布局：

- 左侧树形导航展示企业、部门、用户组和个人。
- 右侧配置面板展示当前节点配额、已使用、剩余额度、预警阈值和继承关系。
- 支持按节点名称搜索定位。
- 支持批量调整同层级节点配额。
- 支持配置日、周、月、季度、年度配额周期。

配置字段包括：

| 字段 | 说明 |
| --- | --- |
| 配额对象 | 企业、部门、用户组或个人 |
| 总 Token 配额 | 当前周期可用总量 |
| 已用 Token | 当前周期已消耗量 |
| 剩余 Token | 当前周期剩余量 |
| 预警阈值 | 达到阈值后触发提醒 |
| 生效周期 | 日、周、月、季度、年度 |
| 超额策略 | 阻断、审批后继续、降级模型、仅提醒 |
| 调整原因 | 管理员填写，写入审计日志 |

#### 7.4.4 配额监控面板

配额监控面板采用仪表盘展示：

- 展示企业、部门、用户组、个人各维度配额使用占比。

- 支持按日、周、月切换趋势图。
- 预警阈值在图表中明确标注。
- 超过预警阈值显示黄色警示。
- 配额用尽或超额显示红色警示。

核心指标包括：

| 指标 | 说明 |
| --- | --- |
| 总配额 | 当前维度总 Token 数 |
| 已使用 | 当前周期消耗 Token 数 |
| 使用率 | 已使用 / 总配额 |
| 日均消耗 | 当前周期内平均每日消耗 |
| 预计耗尽时间 | 基于近期消耗速度预测 |
| 高消耗应用 | Token 消耗最高的应用 |
| 高消耗用户 | Token 消耗最高的用户 |

#### 7.4.5 配额日志

配额日志采用时间轴展示配额相关事件：

- 配额调整记录。
- 预警触发记录。
- 配额用尽记录。
- 超额申请和审批记录。

- 事件类型：调整、预警、用尽、审批、重置。
- 对象类型：企业、部门、用户组、个人。
- 操作人。
- 时间范围。

| 指标 | 展示方式 | 异常规则 |

| CPU 使用率 | 仪表盘、趋势图 | 超过阈值黄/红警示 |
| 内存使用率 | 仪表盘、趋势图 | 超过阈值黄/红警示 |

资源指标包括：

- 内存使用率。

## 8. 非功能需求

### 8.1 安全性

- 密码必须加密存储，不得明文保存。
- 登录态使用安全 Token 机制，并支持过期刷新。
- 所有后台接口必须进行服务端鉴权。
- 敏感字段如密码、Token、密钥、手机号、邮箱需脱敏展示。
- 操作日志和审计日志不可由普通管理员删除。

### 8.2 性能

- 首页首屏加载时间目标小于 3 秒。
- 权限树在 1000 个权限节点内应保持流畅操作。
- 日志查询默认分页加载，避免一次性拉取大量数据。
- 监控图表需支持定时刷新，避免全量刷新造成页面卡顿。

### 8.3 可用性

- 关键操作需有明确确认，例如删除角色、停用用户、调整配额。
- 表单需提供即时校验和明确错误提示。
- 管理页面需支持空状态、加载态、失败态。
- 权限不足时需显示友好提示，不应出现空白页。

### 8.4 可审计性

以下操作必须写入审计日志：

- 登录成功、登录失败、退出登录。
- 新增、编辑、删除、停用用户。
- 新增、编辑、删除、停用角色。
- 权限分配和权限移除。
- Token 配额调整。
- 预警阈值调整。
- 系统配置变更。
- 日志导出。

### 8.5 兼容性

- 支持主流 Chromium 内核浏览器。
- 支持 1366px 及以上桌面分辨率。
- 登录页和门户首页需适配移动端。
- 管理后台至少适配平板和桌面端。

### 8.6 界面美观与一致性

- 整体视觉要求大气、简洁、企业级，信息层级清晰，不追求过多装饰效果。
- 保留原项目中已可用的首页结构和交互，包括顶部导航、Hero 展示、应用矩阵、分类筛选、应用搜索、应用卡片跳转和平台价值区。
- 顶部导航不堆叠过多后台管理入口，权限管理、配额中心、监控运维等能力统一收纳到右上角用户功能下拉中。
- 首页优先突出平台定位和 AI 应用入口，管理能力作为登录后用户操作入口出现，避免喧宾夺主。
- 页面布局保持留白充足，模块间距统一，卡片数量和信息密度适中，避免页面显得拥挤杂乱。
- 色彩以企业品牌主色为核心，搭配中性色和少量状态色；红色、黄色仅用于异常、预警和风险提示。
- 卡片、按钮、表格、弹窗、下拉菜单需沿用 Ant Design 风格，并与当前项目的主色、圆角和字体保持一致。
- 图表和监控数据应优先表达关键结论，避免同屏堆砌过多图表。
- 管理后台页面应采用稳定、克制的工作台风格，强调可读性、可操作性和快速定位问题。

## 9. 权限模型建议

### 9.1 RBAC 模型

平台建议采用 RBAC 权限模型：

```text
用户 -> 用户组 -> 角色 -> 权限
用户 -> 角色 -> 权限
```

企业、部门、用户组作为数据边界和配额边界，角色作为功能权限集合，权限作为最小可控单元。

### 9.2 权限编码规范

权限编码建议采用以下格式：

```text
模块:功能:操作
```

示例：

```text
app:center:view
app:center:access
user:manage:create
user:manage:delete
role:manage:assign_permission
quota:config:update
monitor:log:export
```

### 9.3 数据权限

除功能权限外，系统需支持数据权限：

- 本人数据。
- 本部门数据。
- 本企业数据。
- 跨企业数据。
- 全平台数据。

企业管理员默认仅具备本企业数据权限，超级管理员具备全平台数据权限。

## 11. 关键流程

### 11.1 登录流程

```text
用户进入登录页
-> 选择认证方式
-> 输入认证信息
-> 系统校验身份
-> 获取用户角色和权限
-> 进入门户首页
-> 根据权限展示可访问应用和菜单
```

### 11.2 权限分配流程

```text
管理员进入角色管理页
-> 选择角色
-> 勾选权限树
-> 系统执行冲突检测
-> 展示变更摘要
-> 管理员确认保存
-> 写入权限变更记录
-> 权限实时生效
```

### 11.3 Token 配额校验流程

```text
用户访问 AI 应用
-> 系统校验登录态和应用权限
-> 查询用户及上级组织配额
-> 判断剩余 Token 是否充足
-> 允许调用或阻断调用
-> 记录 Token 消耗
-> 达到阈值时触发预警
```

### 11.4 日志查询流程

```text
运维人员进入日志页
-> 设置级别、关键词、时间范围等筛选条件
-> 查询日志列表
-> 查看 ERROR 日志详情
-> 根据请求 ID 追踪调用链
-> 定位异常服务或接口
```

## 15. 风险与约束

- 当前项目主要为前端门户雏形，缺少后端接口、认证服务、权限服务和监控数据源。
- 角色权限和 Token 配额均需要后端强校验，前端展示不能作为安全边界。
- 监控与日志依赖实际运维体系，需要明确指标采集来源和日志存储方案。
- README 和部分源码中文存在编码显示异常，后续维护时建议统一文件编码为 UTF-8。
- 若需要接入企业 SSO，需提前确认协议类型，如 OAuth2、OIDC、SAML 或企业微信、钉钉等认证体系。
